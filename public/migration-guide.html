Firebase Firestore Setup Guide

Follow these steps to correctly set up your database and fix connection errors.

Step 1: Open Your Firebase Project

Go to the Firebase Console and select the project you are using for this application.

Step 2: Go to Firestore Database

In the left-hand menu, under "Build", click on Firestore Database.

Step 3: Create Database

If you see a "Create database" button, click it. This means you haven't created a database yet, which is the cause of the error.

Step 4: Start in Test Mode

When asked, select "Start in test mode". This sets up security rules that allow your app to read and write data. Click "Next".

Note: Test mode rules are only valid for 30 days. For a production app, you would need to set up more secure rules.

Step 5: Select Location

You will be prompted to select a location for your database. Choose a location close to your users. Any location will work for now. Click "Enable".

Step 6: Verify and Update Security Rules

After the database is created, go to the "Rules" tab at the top.

Step 7: Edit and Publish Rules

Ensure your rules look like the code block below. This rule allows anyone to read and write to your database. Make your changes and click "Publish".

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

Step 8: Refresh Your Application

Once you have published the rules, go back to your application and refresh the page. The error should now be resolved.
