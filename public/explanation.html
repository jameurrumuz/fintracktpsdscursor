Firebase Admin Restricted Operation Error

Learn how to fix the "auth/admin-restricted-operation" error in your Firebase application

Understanding the Error

The FirebaseError: auth/admin-restricted-operation typically occurs when Firebase Authentication is configured to restrict certain operations. This often happens when the Anonymous sign-in provider is disabled in your Firebase project settings.

// Example error message:
FirebaseError: Firebase: Error (auth/admin-restricted-operation).
at createErrorInternal (https://...firebase-auth.js:542:41)
at _fail (https://...firebase-auth.js:507:11)
at _performFetchWithErrorHandling (https://...firebase-auth.js:968:17)

This error can also occur due to incorrect security rules, API restrictions, or domain authorization issues.

Solution 1: Enable Anonymous Sign-in

The most common solution is to enable Anonymous authentication in your Firebase project:

  1. Go to the Firebase Console
  2. Select your project
  3. Navigate to Authentication → Sign-in method
  4. Find the Anonymous provider and click on it
  5. Enable Anonymous sign-in and click Save

Important: After making changes in the Firebase Console, it may take a few minutes for the changes to propagate.

Solution 2: Verify Your Implementation

Ensure your code follows the correct implementation for anonymous authentication:

// Import the necessary Firebase modules
import { initializeApp } from "firebase/app";
import { getAuth, signInAnonymously } from "firebase/auth";

// Your Firebase configuration
const firebaseConfig = {
  apiKey: "your-api-key",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project.appspot.com",
  messagingSenderId: "123456789",
  appId: "your-app-id"
};

// Initialize Firebase
const app = initializeApp(firebaseConfig);
const auth = getAuth(app);

// Sign in anonymously
signInAnonymously(auth)
  .then((userCredential) => {
    // Signed in successfully
    const user = userCredential.user;
    console.log("Anonymous user ID:", user.uid);
  })
  .catch((error) => {
    const errorCode = error.code;
    const errorMessage = error.message;
    console.error("Authentication error:", errorCode, errorMessage);
  });

Make sure your Firebase configuration object is correct and matches your project settings.

Solution 3: Check Additional Settings

Authorized Domains

Ensure your domain is authorized in the Firebase Console:

  1. Go to Firebase Console → Authentication → Settings
  2. Under "Authorized domains", make sure your domain is listed
  3. Add any missing domains (localhost is included by default for testing)

API Restrictions

Check if there are any API restrictions on your API key:

  1. Go to the Google Cloud Console
  2. Select your project
  3. Navigate to APIs & Services → Credentials
  4. Find your API key and check if there are any restrictions that might be blocking authentication

Security Rules

Check your Firebase Security Rules to ensure they allow the operations you're trying to perform.

Note: If you're testing locally, make sure you're using http://localhost or http://127.0.0.1 as these are authorized by default in Firebase projects.

Still Need Help?

If you're still experiencing the issue after trying these solutions:

  1. Check the Firebase Status Dashboard for any ongoing incidents
  2. Make sure you're using the latest version of the Firebase SDK
  3. Try testing in incognito mode or a different browser to rule out extension conflicts
  4. Check your browser's console for any additional error messages

For more information, visit the Firebase Authentication documentation.
